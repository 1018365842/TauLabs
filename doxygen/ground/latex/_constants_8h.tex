\subsubsection[{Dynamic}]{\setlength{\rightskip}{0pt plus 5cm}const {\bf int} Dynamic = 10000}}\label{_constants_8h_adc9da5be31bdce40c25a92c27999c0e3}
This value means that a quantity is not known at compile-\/time, and that instead the value is stored in some runtime variable.

Explanation for the choice of this value\-:
\begin{DoxyItemize}
\item It should be positive and larger than any reasonable compile-\/time-\/fixed number of rows or columns. This allows to simplify many compile-\/time conditions throughout \hyperlink{namespace_eigen}{Eigen}.
\item It should be smaller than the sqrt of I\-N\-T\-\_\-\-M\-A\-X. Indeed, we often multiply a number of rows with a number of columns in order to compute a number of coefficients. Even if we guard that with an \char`\"{}if\char`\"{} checking whether the values are Dynamic, we still get a compiler warning \char`\"{}integer overflow\char`\"{}. So the only way to get around it would be a meta-\/selector. Doing this everywhere would reduce code readability and lenghten compilation times. Also, disabling compiler warnings for integer overflow, sounds like a bad idea.
\end{DoxyItemize}

If you wish to port \hyperlink{namespace_eigen}{Eigen} to a platform where sizeof(int)==2, it is perfectly possible to set Dynamic to, say, 100.
