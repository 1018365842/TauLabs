\subsection{Detailed Description}
\subsubsection*{template$<$class T$>$class Qxt\-Bdb\-Tree$<$ T $>$}

The \hyperlink{class_qxt_bdb_tree}{Qxt\-Bdb\-Tree} class is a template berkeley container for tree structured data. 

Qxt\-Berkeley The template argument must be registered with the Qt meta system. You may not touch the file while a \hyperlink{class_qxt_bdb_tree}{Qxt\-Bdb\-Tree} instance is running on it.

Example usage\-: 
\begin{DoxyCode}
\hyperlink{class_qxt_bdb_tree}{QxtBdbTree<QString>} db(\textcolor{stringliteral}{"test.db"});
db.root().append(\textcolor{stringliteral}{"fooooo"}).append(\textcolor{stringliteral}{"bla"});
db.dumpTree(); \textcolor{comment}{//try this if you are unsure, how the data will look like}
\end{DoxyCode}


There is an extensive example in /examples/berkeley/xmlstorage

All functions of this class are thread safe. Calling \hyperlink{class_qxt_bdb_tree_a32daaad953219b5bf3c5620711e7159b}{open()} multiple times is undefined. An iterator may only be used from one thread at once, but you can have multiple iterators.
