\subsection{Detailed Description}
The \hyperlink{class_qxt_multi_signal_waiter}{Qxt\-Multi\-Signal\-Waiter} class blocks and processes events until a group of signals is emitted. 

Qxt\-Core

Code written in a synchronous style will sometimes need to block until any of several conditions are met, or until all of a set of conditions are met. This class allows a group of signals to be defined and waited upon in a simple A\-N\-D or O\-R fashion. More complex Boolean relationships can be written by connecting together multiple \hyperlink{class_qxt_signal_group}{Qxt\-Signal\-Group} objects and waiting for all or any of these groups to emit one of their signals.

\{Note\-:\} \hyperlink{class_qxt_multi_signal_waiter}{Qxt\-Multi\-Signal\-Waiter} is subject to the same reentrancy problems as \hyperlink{class_qxt_signal_waiter}{Qxt\-Signal\-Waiter}.

Blocks the current function by processing events from the event queue according to {\itshape flags} until all of the signals in the group have been emitted. If {\itshape msec} is not {\ttfamily -\/1}, \hyperlink{class_qxt_multi_signal_waiter_a6cb02fd64ef57ab7bbf6a5e81c4147c9}{wait\-For\-All()} will return before all of the signals are emitted if the specified number of milliseconds have elapsed. Returns {\ttfamily true} if each signal was caught at least once, or {\ttfamily false} if the timeout elapsed. Note that \hyperlink{class_qxt_multi_signal_waiter_a6cb02fd64ef57ab7bbf6a5e81c4147c9}{wait\-For\-All()} may continue to block after the last signal is emitted or the timeout elapses; the function only guarantees that it will not return B\-E\-F\-O\-R\-E one of these conditions has occurred. This function is not reentrant.
