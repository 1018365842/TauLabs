\subsection{Detailed Description}
The \hyperlink{class_qxt_r_p_c_peer}{Qxt\-R\-P\-C\-Peer} class transmits Qt signals over a network connection. 

Qxt\-Network \hyperlink{class_qxt_r_p_c_peer}{Qxt\-R\-P\-C\-Peer} is a tool that encapsulates Qt signals and transmits them over a network connection. The signal is subsequently re-\/emitted on the receiving end of the connection.

As a \hyperlink{class_qxt_r_p_c_service}{Qxt\-R\-P\-C\-Service} subclass, \hyperlink{class_qxt_r_p_c_peer}{Qxt\-R\-P\-C\-Peer} can act as a client or a server. When acting as a server, it uses a \hyperlink{class_qxt_tcp_connection_manager}{Qxt\-Tcp\-Connection\-Manager} to accept connections. When acting as a client, applications may use \hyperlink{class_qxt_r_p_c_peer_ae0c492f74829b363831bb4ce75a1d21f}{connect()} to establish a T\-C\-P connection with a server.

All data types used in attached signals and slots must be declared and registered with Q\-Meta\-Type using Q\-\_\-\-D\-E\-C\-L\-A\-R\-E\-\_\-\-M\-E\-T\-A\-T\-Y\-P\-E and q\-Register\-Meta\-Type. Additional requirements may be imposed by the \hyperlink{class_qxt_abstract_signal_serializer}{Qxt\-Abstract\-Signal\-Serializer} subclass in use; the default \hyperlink{class_qxt_data_stream_signal_serializer}{Qxt\-Data\-Stream\-Signal\-Serializer} requires that they have stream operators registered with q\-Register\-Meta\-Type\-Stream\-Operators.

Due to a restriction of Qt's signals and slots mechanism, the number of parameters that can be passed to \hyperlink{class_qxt_r_p_c_service_a5a0cb803d5f0dbadc102cc33cef612a3}{call()} and its related functions, as well as the number of parameters to any signal or slot attached to \hyperlink{class_qxt_r_p_c_peer}{Qxt\-R\-P\-C\-Peer}, is limited to 8.

Listens on the specified interface {\itshape iface} on the specified {\itshape port} for connections.

Attempting to listen while in Client mode or while connected in Peer mode will be ignored with a warning. In Peer mode, only one connection can be active at a time. Additional incoming connections while connected to a peer will be dropped. When a peer connects, the peer\-Connected() signal will be emitted. In Server mode, multiple connections can be active at a time. Each client that connects will be provided a unique I\-D, included in the \hyperlink{class_qxt_r_p_c_service_a9b2af33db0cebccc82ecc2b9d21c8843}{client\-Connected()} signal that will be emitted. Returns {\ttfamily true} if succeeds, {\ttfamily false} otherwise.
