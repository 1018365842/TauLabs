51 /* The submit editor presents the commit message in a text editor and an
52  * checkable list of modified files in a list window. The user can delete
53  * files from the list by unchecking them or diff the selection
54  * by doubleclicking. A list model which contains the file in a column
55  * specified by fileNameColumn should be set using setFileModel().
56  *
57  * Additionally, standard creator actions can be registered:
58  * Undo/redo will be set up to work with the description editor.
59  * Submit will be set up to be enabled according to checkstate.
60  * Diff will be set up to trigger diffSelected().
61  *
62  * Note that the actions are connected by signals; in the rare event that there
63  * are several instances of the SubmitEditorWidget belonging to the same
64  * context active, the actions must be registered/unregistered in the editor
65  * change event.
66  * Care should be taken to ensure the widget is deleted properly when the
67  * editor closes. */
