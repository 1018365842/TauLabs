1203  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS ONLY
1204  * INTENDED FOR USE WHEN IMPLEMENTING A PORT OF THE SCHEDULER AND IS
1205  * AN INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1206  *
1207  * Called from the real time kernel tick (either preemptive or cooperative),

1210  * a ready list.

1214 /*
1215  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN
1216  * INTERFACE WHICH IS FOR THE EXCLUSIVE USE OF THE SCHEDULER.
1217  *
1218  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
1219  *
1220  * Removes the calling task from the ready list and places it both
1221  * on the list of tasks waiting for a particular event, and the
1222  * list of delayed tasks. The task will be removed from both lists
1223  * and replaced on the ready list should either the event occur (and
1224  * there be no higher priority tasks waiting on the same event) or
1225  * the delay period expires.
1226  *
1227  * @param pxEventList The list containing tasks that are blocked waiting
1228  * for the event to occur.
1229  *
1230  * @param xTicksToWait The maximum amount of time that the task should wait
1231  * for the event to occur. This is specified in kernel ticks,the constant
1232  * portTICK_RATE_MS can be used to convert kernel ticks into a real time
1233  * period.
1234  */
1235 void vTaskPlaceOnEventList( const xList * const pxEventList, portTickType xTicksToWait ) PRIVILEGED_FUNCTION;

1237 /*
1238  * THIS FUNCTION MUST NOT BE USED FROM APPLICATION CODE. IT IS AN

1241  * THIS FUNCTION MUST BE CALLED WITH INTERRUPTS DISABLED.
